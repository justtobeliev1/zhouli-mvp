# Zhouli Card Skill

## 何时使用
- 用户要把一句现代吐槽改写成古风判词。
- 用户要可传播文案：真引/意译/戏仿三连。
- 用户要“代码生成卡片”，强调稳定导出与固定风格。
- 用户要在自己的 agent 中复用该流程。

## 输入
- 一句现代处境（必填）。
- 可选：语气强度（温和/标准/更毒）。
- 可选：输出用途（评论区/标题/图卡）。

## 标准输出结构
- 先生成结构化 JSON（字段固定，便于稳定渲染）。
- 再生成 `markdown`（给聊天界面渲染）。
- 最后由代码模板绘制卡片（Canvas/SVG/HTML 均可，优先现有模板）。

读取并遵守 [card_schema.json](references/card_schema.json)。

## 真引规则（强约束）
- 若存在可核验真实古文原句：必须优先使用原句 + 真实出处。
- 若无法确认真实出处：`true.empty=true`，`true.title="暂缺可信原句"`，`true.source=""`。
- 不允许编造书名或篇名。

## 文案风格
- 一本正经引经据典，夹带机锋和轻度戏谑。
- 句子短，梗密度高，可截图传播。
- 可以讽刺行为，不做人身侮辱，不用脏话，不涉仇恨歧视。

系统提示词模板在 [system_prompt.txt](references/system_prompt.txt)。

## 执行流程
1. 读取用户输入，判断主题（室友/小组/DDL/社交等）。
2. 调用 LLM，严格返回 `card_schema.json` 对应结构。
3. 用 `scripts/validate_card_payload.js` 校验字段与长度。
4. 校验失败时进行一次修复重试；仍失败则降级 mock。
5. 将最终 payload 交给代码渲染层导出图卡（固定尺寸建议 1080x1350）。

## 输出质量门槛
- 字段完整、类型正确、长度合规。
- 真引规则满足，来源可信或明确降级。
- 卡片不溢出、对比度达标、导出稳定。
